concepto de git

Git es un sistema de control de versiones nos permite llevar un historial de todos los cambios que existan en un proyecto, sirve para documentar y trazar cada uno de los cambios, en pocas palabras es una "fotocopiadora" de archivos que registra cada cambio que se hace en un proyecto.

Principales comandos de Git

  • GIT INIT: Este comando sirve para inicializar un proyecto con git.
  • GIT ADD: Permite agregar todos los archivos modificados y nuevos al area de preparación para ser incluidos en el siguiente commit.
  • GIT COMMIT -M "": Crea un nuevo commit con los cambios en el area de preparación y lo guarda en el historial de versiones del repositorio
  • GIT CONFIG --GLOBAL USER.NAME: Se utiliza para configurar el nombre del usuario de forma global.
  • GIT BRANCH: Sirve para ver todas las ramas en el repositorio y resalta la rama actual.
  • GIT CONFIG --GLOBAL USER.EMAIL: Se utiliza para configurar el email de forma global.
  • GIT PUSH: Envia tus commits locales al repositorio correspondiente.
  • GIT LOG: Mestra el historial de commits realizados en el repositorio, incluyendo detalles como el autor, la fecha y el mensaje del commit.
  • GIT CHECKOUT MAIN: Cuando por fuera de la rama main me premite ingresar a ella.
  • GIT REMOTE ADD ORIGIN: Para conectar con github

Resultado poder iniciar sesión o registrarse para navegar
Flujo normal
  1. El usuario visita la pagina y hace clic en el boton iniciar sesión.
  2. El usuario completa el formulario de registro, que incluye su nombre, dirección de correo electrónico y una contraseña segur o en caso de iniciar sesión solamente correo y contraseña.
  3. Airbnb envía un correo electrónico de verificación a la dirección de correo electrónico proporcionada por el usuario.
  4. El usuario hace clic en el enlace de verificación en el correo electrónico para confirmar su dirección de correo electrónico.
  5. El usuario es redirigido a la página de inicio de sesión de Airbnb y puede iniciar sesión en su cuenta recién creada.
Criterios de aceptación
  1. El formulario de registro debe ser fácil de completar y no debe requerir información innecesaria.
  2. El correo electrónico de verificación debe enviarse de manera oportuna y contener un enlace de verificación válido.
  3. El proceso de verificación de correo electrónico debe ser fácil de entender y completar.
  4. Los usuarios deben poder iniciar sesión en su cuenta recién creada sin problemas.
  5. La contraseña debe cumplir con los requisitos de seguridad de Airbnb y ser fácil de restablecer si el usuario la olvida.
