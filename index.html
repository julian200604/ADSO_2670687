concepto de git

Git es un sistema de control de versiones nos permite llevar un historial de todos los cambios que existan en un proyecto, sirve para documentar y trazar cada uno de los cambios, en pocas palabras es una "fotocopiadora" de archivos que registra cada cambio que se hace en un proyecto.

Principales comandos de Git

  • GIT INIT: Este comando sirve para inicializar un proyecto con git.
  • GIT ADD: Permite agregar todos los archivos modificados y nuevos al area de preparación para ser incluidos en el siguiente commit.
  • GIT COMMIT -M "": Crea un nuevo commit con los cambios en el area de preparación y lo guarda en el historial de versiones del repositorio
  • GIT CONFIG --GLOBAL USER.NAME: Se utiliza para configurar el nombre del usuario de forma global.
  • GIT BRANCH: Sirve para ver todas las ramas en el repositorio y resalta la rama actual.
  • GIT CONFIG --GLOBAL USER.EMAIL: Se utiliza para configurar el email de forma global.
  • GIT PUSH: Envia tus commits locales al repositorio correspondiente.
  • GIT LOG: Mestra el historial de commits realizados en el repositorio, incluyendo detalles como el autor, la fecha y el mensaje del commit.
  • GIT CHECKOUT MAIN: Cuando por fuera de la rama main me premite ingresar a ella.
  • GIT REMOTE ADD ORIGIN: Para conectar con github
